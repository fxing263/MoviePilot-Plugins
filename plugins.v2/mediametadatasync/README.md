# 媒体元数据双目录同步

MoviePilot V2 插件，用于在 `/media/9kg` 与 `/media/番号系列` 之间同步 STRM 媒体库元数据。

管理页和配置页内置独立的亮色、暗色高对比配色，避免宿主自定义主题使文字或禁用控件不可见。Vue 联邦资源按插件版本目录发布，升级后不会继续复用旧版界面缓存。

## 目录约定

```text
/media/9kg/名字或合集名/番号目录/元数据文件
/media/番号系列/A-Z或其他/番号目录/元数据文件
```

番号目录名以英文字母开头时进入对应的大写 `A-Z` 分类，否则进入 `其他`。

## 同步行为

- 正向全量：扫描所有 `名字或合集名/番号目录`，同步配置扩展名内的文件，不受实时就绪条件限制。
- 正向实时：未激活目录只有出现同名 `.strm` 与 `.nfo` 后才会首次同步；激活后继续同步图片、NFO、STRM 和 JSON 变更。
- 反向全量：扫描 `番号系列/*/番号目录/*.json`，回写到 9kg 内全部同名番号目录。
- 反向实时：只处理番号系列中新增或修改的 `.json`。
- 插件启用时先在后台完成双向全量同步，再按配置启动实时监控。

复制使用同目录临时文件和原子替换。目标文件大小与修改时间相同会跳过；双向 JSON 冲突保留修改时间较新的版本，其他正向元数据以 9kg 为准。同步任务不传播删除，也不自动清理孤儿目录。

## 缺失元数据检查

- 后台扫描 9kg 中的全部 STRM，检查同名 `.nfo` 与 `<文件名>-mediainfo.json`。
- 同一番号和同一 STRM 文件名在 9kg 中存在多个位置时，聚合为一条记录。
- 结果支持分页和搜索；每条记录都可以生成精确删除预览。

手动删除固定按以下顺序执行：

1. 删除 STRM 指向的网盘挂载源文件，并通过父目录重新列举确认文件已不存在。
2. 删除番号系列中的该 STRM 及其独占元数据，并复核结果。
3. 删除 9kg 中全部同番号、同文件名的位置及其独占元数据，并复核结果。

删除前必须先生成三端文件清单并获得绑定当前文件大小和修改时间的确认令牌。文件状态发生变化时令牌立即失效；任一阶段复核失败都会停止后续阶段。只删除普通文件，不删除目录或符号链接。

## 默认配置

| 配置 | 默认值 |
| --- | --- |
| 源目录 | `/media/9kg` |
| 目标目录 | `/media/番号系列` |
| 其他分类 | `其他` |
| 事件沉淀时间 | `2` 秒 |
| 正向扩展名 | `.strm,.nfo,.jpg,.jpeg,.png,.webp,.gif,.json` |
| JSON 实时回写 | 启用 |
| 网盘挂载根目录 | `/CloudNAS/CloudDrive` |
| 单条最大删除文件数 | `100` |

插件详情页可提交双向、正向、JSON 回写和缺失元数据扫描任务，并轮询展示后台任务、缺失记录、删除复核、待就绪目录、分类统计和最近日志。
