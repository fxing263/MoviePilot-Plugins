# 三插件统一运维工作台 UI 设计

## 目标

将以下三个 Vue/Vuetify 插件管理页统一为高密度、易扫描、响应式的运维工作台，同时保持现有业务接口、任务状态机和删除安全链不变：

- `EmbyLibraryOrganizer`（Emby 媒体库整理）
- `MediaMetadataSync`（媒体元数据双目录同步）
- `DirectoryFileSearch`（目录文件搜索删除）

权威源码位于 `/home/zhaojg/project/MoviePilot-Plugins`。`/home/zhaojg/MoviePilot/app/plugins` 只作为 MoviePilot 运行时测试副本，不在源码修改阶段直接编辑。

## 范围

本次改造覆盖三个插件的 `Page.vue`、`Config.vue` 和必要的本地预览数据。允许调整 Vue 模板、派生展示状态和局部 CSS；不新增后端接口，不改变接口路径、请求参数、响应结构、扫描行为、删除确认令牌或复核流程。

不引入新的前端依赖，不修改 MoviePilot 主仓库的公共组件，不抽取跨插件共享包。三个插件保留独立构建产物，以避免联邦模块之间产生新的运行时耦合。

## 统一信息架构

### 页面骨架

三个主页面采用相同的纵向结构：

1. 页头：插件名称、状态胶囊、当前上下文和刷新/设置/关闭操作。
2. 任务反馈区：为排队、运行、失败和完成反馈预留稳定区域，避免异步状态引发布局跳动。
3. 指标状态带：四个可快速扫描的数字或时间指标，数字使用等宽数字样式。
4. 视图标签：只承载插件现有的一级工作区，不新增导航层级。
5. 当前视图工具栏：筛选或任务范围位于左侧，当前视图唯一主操作位于右侧。
6. 结果区域：桌面使用紧凑表格或分组列表，移动端转换为信息行。
7. 批量操作条：仅在有可操作结果或已有选择时出现，危险操作与普通操作分离。
8. 分页或活动摘要：保留现有分页与最近任务信息。

### 插件映射

`EmbyLibraryOrganizer`：

- 指标为缺失元数据数、多余元数据数、可用分类数和最近完成时间。
- 保留“缺失元数据 / 多余元数据”标签。
- 两类扫描范围分别收进对应标签的工具栏，避免页面顶部同时堆叠两个主按钮。
- 缺失项保留分组展开和四种既有删除动作；多余项保留当前页选择与批量删除。

`MediaMetadataSync`：

- 指标为扫描目录、就绪目录、复制数量和待处理数量。
- 保留“同步 / 缺失元数据 / 运行记录”标签。
- 同步页只突出“双向全量”为主操作，正向全量、JSON 回写和监控启停保持次级层级。
- 缺失元数据页保留检索、分页和三端删除预览。

`DirectoryFileSearch`：

- 指标为已扫描文件、匹配结果、不可访问数量和完成时间。
- 工作区统一为结果、任务状态与执行记录的层级，但不新增后端数据要求；缺失数据由现有状态和报告映射。
- 保留搜索、当前页选择、全部结果选择、单条删除和批量删除。

## 视觉系统

优先使用 Vuetify 的宿主语义 token，包括 `surface`、`on-surface`、`primary`、`success`、`warning`、`error` 和边框变量。局部 CSS token 仅用于统一间距、尺寸和透明度，不强制插件进入独立亮色或暗色主题。

采用 4/8px 间距节奏，主要区块间距为 16px 或 24px。卡片圆角不超过 8px；页面区段保持无框或使用分隔线，不把卡片嵌套在卡片中。正文沿用宿主字体，字间距保持 `0`。

交互动画限制为 150-220ms 的颜色、透明度和轻微阴影变化，不动画宽高或位置。`prefers-reduced-motion` 下关闭非必要过渡。

## 交互与状态

- 图标按钮保持一致尺寸，并提供可访问名称；可点击区域不小于 44px。
- 所有异步按钮在执行时显示加载状态并禁用重复提交。
- 禁用状态必须同时具有语义属性和清晰视觉差异。
- 状态不能只靠颜色表达，必须包含文字、图标或形状。
- 初始加载、任务运行、失败、插件未启用、未配置、空结果和成功完成均使用明确状态文案。
- 危险操作只使用 `error` 语义色，且始终经过现有预览、勾选确认、确认令牌和执行复核流程。
- 成功或失败提示继续使用 snackbar；错误文案应说明原因或恢复路径。

## 响应式规则

- 设计基准视口为 375、768、1024 和 1440px。
- 760px 以下页头允许换行，指标状态带改为两列。
- 桌面结果表在移动端转换为三列信息行：选择、主信息、行操作；路径、大小和时间折叠进次级文案。
- 工具栏在移动端改为单列，主按钮占满可用宽度。
- 批量操作条允许换行，不使用覆盖内容的固定定位。
- 页面不得产生视口级横向滚动；长路径在可控容器内截断，并通过标题或展开方式提供完整内容。
- 配置页使用可滚动内容区和稳定底部操作栏，内容不得被底部操作遮挡。

## 配置页

三个配置页统一为页头、分组内容和底部操作栏：

- 运行开关单独成组。
- 路径、分类或同步规则按业务关系分组。
- 删除边界与危险设置使用警告说明，但不使用装饰性大卡片。
- 输入保留可见标签；复杂字段提供持续可见的辅助文字。
- 校验错误显示在对应字段附近，保存按钮在无效时禁用。
- 取消与保存的顺序、尺寸和层级保持一致。

## 架构与代码约束

本次只修改插件前端展示层和构建产物，不改变 MoviePilot 的 Entrypoint、Chain、Module、Helper 或 Oper 依赖方向。不会新增 Python 公共接口，因此中文 docstring 门槛不产生新覆盖面；若实现中确需修改 Python 公共类或方法，必须补齐中文 docstring 和类型标注后再提交。

文件名、组件名和现有插件标识保持不变。不会新增数据库模型、系统配置键、REST/MCP 接口或外部 HTTP 请求，因此不需要 Alembic、`SystemConfigKey`、API 文档或安全扫描联动更新。

## 验证

实现完成后执行以下验证：

1. 分别在三个 `frontend/<plugin>` 目录执行现有 Vite 构建。
2. 运行三个插件对应的 pytest 测试，确认接口契约和任务安全链未被 UI 改动影响。
3. 如 Python 文件未修改，记录 pylint 不适用；如有修改，运行对应模块的 pylint。
4. 在桌面和移动视口检查主页面与配置页，覆盖亮色、暗色、加载、空结果、失败、禁用、批量选择和删除确认状态。
5. 检查键盘焦点、图标按钮名称、44px 触控目标、文本对比度、长路径换行或截断，以及 `prefers-reduced-motion`。
6. 将三个插件的 Python 文件和构建后的 `dist` 同步到 `/home/zhaojg/MoviePilot/app/plugins/<plugin>`。
7. 使用文件比较确认权威源码与运行时副本一致；重启 MoviePilot 并确认三个插件加载版本与页面资源正常。

## 成功标准

- 三个插件看起来属于同一套 MoviePilot 运维工具，而不是三个独立设计。
- 高频扫描、筛选、批量选择和结果复核比当前页面更易扫描，且不增加必要点击次数。
- 危险操作更明确，但删除流程、确认令牌和保护边界不被削弱。
- 375px 到 1440px 无页面级横向滚动、内容遮挡或控件重叠。
- 亮暗主题均保持清晰文本、边框、禁用态和状态反馈。
- 所有既有前端构建和目标 pytest 测试通过，运行时副本与权威源码一致。
