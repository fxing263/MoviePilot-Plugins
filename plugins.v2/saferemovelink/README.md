# 清理媒体文件（115安全队列版）

SafeRemoveLink 基于清理媒体文件插件的功能建立独立源码，在 STRM 删除触发 115 网盘清理时使用持久化单消费者队列，避免批量事件并发删除导致访问上限。

## 安装

将以下地址加入 MoviePilot 的 `PLUGIN_MARKET`，刷新插件市场后搜索“清理媒体文件（115安全队列版）”：

```text
https://github.com/fxing263/MoviePilot-Plugins
```

插件安装后默认关闭。启用前请先停用原 `RemoveLink`，再检查监控目录和 STRM 路径映射。

## 使用约束

- SafeRemoveLink 与原 `RemoveLink` 不能同时监听相同的硬链接或 STRM 目录。启用本插件前应停用原插件，或移除原插件中的相同路径映射。
- 插件安装后默认关闭，不会自动执行删除。确认监控目录、排除目录和 STRM 路径映射后再启用。
- 115 删除仍通过 MoviePilot 的 `StorageChain` 执行，只移动到 115 回收站，不直接调用第三方 HTTP API。

## 队列行为

- 115 媒体文件、关联刮削文件和空目录共用一个删除槽，默认每次删除至少间隔 30 秒。
- 相同 115 目标路径已有待处理或执行中任务时不会重复入队。
- 首次删除失败后最多自动重试 3 次，退避时间依次为 30、60、120 秒。
- 单任务最终失败后队列冷却 1 小时，再尝试后续任务。
- 连续 3 个任务最终失败后队列暂停并发送通知，需要在配置页开启“恢复暂停的115删除队列”并保存。
- 主媒体删除成功后，即使附属刮削文件或空目录清理失败，也不会重新删除主媒体文件。

## 持久化与重启

待处理任务、失败状态、冷却时间和熔断状态保存到插件数据。MoviePilot 或插件重启后，原先执行中的任务会恢复为待处理并由单消费者继续执行。停止插件时不会强制清空剩余任务。

115 存储模块自身在触发访问上限后可能进入 3600 秒全局冷却；此时当前消费者会等待存储接口返回，队列不会并发启动第二个删除任务。
